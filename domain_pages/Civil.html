CE01

Description In any transportation planning and traffic management studies classified traffic volume is a key input needed. Counting and classifying vehicular traffic manually is very time and resource consuming. Moreover, such manual approaches are not suitable for intelligent transport application. The aim of this project is to use Neural Network Based approaches to detect and classify vehicular traffic. In the first stage, offline video will be used to training and validating results. Once the offline results are satisfactory, the next task is to perform the same task with using real time continuous streaming of videos.
Number of students two
Year of study Students entering their 3rd year, Students entering their 4th/5th year
CPI Above 7.0
Prerequisites Some experience with image processing.
Duration Around 3 months and may be extended.
Learning outcome 1) Understanding road traffic flow, 2) Application of Convolutional Neural Networks, 3) Training of image processing algorithms
Weekly time commitment Around 20 hours a week. Weekly meetings along with graduate students.

CE02

Description To understand and develop instruments for water wave measurement. The project would involve application of engineering principles to create innovative instruments applicable in the field of Ocean Engineering. The instruments include but not limited to wave probes, pressure transducers, velocity measurement devices, Displacement and acceleration. The project would involve use of micro controllers and analogue to digital converters for data aquisition.

Year of study Students entering their 3rd year, Students entering their 4th/5th year

CE03

Description The project aims to create an accessible and user-friendly platform for gathering travel pattern data crucial for travel demand modelling projects. This digital survey tool will be designed to be compatible with various devices, including tablets and mobile phones, ensuring flexibility and convenience for respondents. By leveraging technology, the project seeks to streamline the data collection process, improving efficiency and accuracy in gathering essential travel behaviour information. Through this initiative, stakeholders can access comprehensive and reliable data to inform transportation planning and decision-making processes effectively.
Number of students 4
Year of study Students entering their 3rd year, Students entering their 4th/5th year
CPI NA
Prerequisites Programming proficiency in any coding language and web development experience are required.
Duration Minimum 4 months
Learning outcome 1. Technical Skills Development: Students may learn technical skills related to software development, database management, and user interface design through hands-on experience with the digital survey tool. 2. Data Analysis Skills: Students may gain proficiency in data analysis techniques by working with collected travel pattern data and extracting meaningful insights using analytical tools. 3. Understanding of Transportation Planning: Students may gain insights into transportation planning processes and methodologies by participating in discussions, workshops, and meetings related to travel demand modelling projects.
Weekly time commitment Weekly commitment expected from the student (in hours) = 15 hours/week Frequency of meets the students should have every week = twice/week
Assignment https://www.turing.com/kb/fastapi-vs-flask-a-detailed-comparison https://www.w3schools.com/nodejs/default.asp https://www.w3schools.com/REACT/DEFAULT.ASP https://tailwindcss.com/ https://developers.google.com/maps/documentation/javascript/tutorials https://pypi.org/project/psycopg/ https://www.postgresql.org/ https://www.docker.com/ https://nginx.org/en/
Instructions for assignment Dear Students, You are tasked with understanding the key concepts of web development using the resources provided above. Your goal is to learn from each link and prepare a comprehensive 10-page report detailing how one can develop a web-based survey tool using the technologies and frameworks mentioned. Steps to Follow 1. FastAPI vs Flask: - Understand the differences and use cases for FastAPI and Flask. - Identify which framework is more suitable for the backend of a web-based survey tool. - Summarize the key features of the chosen framework. 2. Node.js: - Learn the basics of Node.js and its importance in building scalable network applications. - Determine how Node.js can be integrated into your survey tool project, especially for backend services. 3. React: - Study the fundamental concepts of React and how it can be used to build dynamic user interfaces. - Explain how React can be utilized to create the frontend of your survey tool. 4. Tailwind CSS: - Explore the utility-first CSS framework and how it helps in designing responsive and modern user interfaces. - Describe how Tailwind CSS can be used to style your survey tool. 5. Google Maps JavaScript API: - Review the tutorials on adding interactive maps to your web application. - Discuss potential uses of Google Maps in a survey tool, such as location-based questions. 6. psycopg and PostgreSQL: - Learn about psycopg as a PostgreSQL adapter for Python and its role in database interactions. - Understand PostgreSQL as a powerful, open-source object-relational database system. - Explain how to set up and interact with a PostgreSQL database for storing survey data. 7. Docker: - Investigate how Docker can be used to containerize your application, making it easier to deploy and manage. - Discuss the benefits of using Docker in development and production environments for your survey tool. 8. NGINX: - Learn about NGINX as a web server that can also be used as a reverse proxy, load balancer, and HTTP cache. - Explain how NGINX can be configured to serve your web-based survey tool efficiently. Report Structure Your report should include the following sections: 1. Introduction: - Briefly introduce the project and the importance of web-based survey tools. 2. Technology Overview: - Provide a summary of each technology and framework based on the resources. - Highlight their roles and significance in web development. 3. Framework and Tools Selection: - Justify the selection of specific tools and frameworks for developing the survey tool. 4. Development Plan: - Outline the steps and methodologies to be followed in developing the survey tool. - Include details on setting up the development environment, backend and frontend development, database integration, and deployment. 5. Implementation Details: - Provide a detailed explanation of how each tool and framework will be used in the project. - Include code snippets, configurations, and setup instructions where applicable. 6. Conclusion: - Summarize the key points of your report. - Discuss potential challenges and future enhancements for the survey tool. Submission Guidelines - The report should be 10 pages long, formatted professionally with clear headings and subheadings. - Include diagrams, code snippets, and screenshots to illustrate your points. - Submit the report as a WORD document. This assignment is designed to give you hands-on experience with modern web development tools and practices. Note: Even if you cannot complete the report at this time, it is crucial that you go through all the concepts provided in the resources before starting work on the project. You are expected to submit the completed report at the end of the project.
